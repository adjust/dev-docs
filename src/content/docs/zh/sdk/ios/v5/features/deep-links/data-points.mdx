在应用中设置深度链接前，请调取下列数据点：

- 应用 ID 前缀
- 发布 Bundle ID
- Debug Bundle ID
- 发布版自定义 URL 方案
- 调试版自定义 URL 方案
- 链接解析域 \(一个或多个\)

## 操作说明 {#instructions}

按照下方说明操作，调取您的数据点。

### 应用 ID 前缀和 Release Bundle ID {#app-id-prefix-and-release-bundle-id}

您可以在 Apple Developer 门户中找到自己的应用 ID。应用 ID 由 2 个部分组成：

1. **应用 ID 前缀**
2. **Bundle ID**

ID 格式为 `<app_id_prefix>.<bundle_id>`。例如： `ABC1234567.com.example.app`

请按照下列步骤操作，找到您的应用 ID 前缀和 Bundle ID：

1. 登入 [Apple Developer 门户](https://developer.apple.com/account/)。
2. 从左侧菜单中选择 **Certificates, IDs & Profiles** 。
3. 从左侧菜单中选择 **Identifiers** 。
4. 找到并选中您的应用，打开编辑界面。
5. 您的应用 ID 前缀 和 Bundle ID 就显示在页面顶端。请复制并保存需要的信息，以备今后使用。

### Debug Bundle ID {#debug-bundle-id}

如果您的调试版本使用了不同的 Bundle ID，请在 Xcode 中找到该 ID。

1. 使用 XCode 打开您的应用项目。
2. 从左侧菜单中选择您的项目。
3. 在 **Targets** 中选择您的应用。
4. 从顶部菜单中选择 **Signing & Capabilities** 。
5. 从弹出的子菜单中选择 **Debug** 。
6. 此时您就会看到 Bundle ID。请复制并保存该信息，以备今后使用。

### 自定义 URL 方案 {#custom-url-schemes}

<Callout type="tip">

若要从设备上的其他应用 \(如 Telegram、X \(前 Twitter\) 或 YouTube\) 或推送通知进行连接，则需要自定义 URL 方案。请咨询您的营销团队，决定是否需要自定义 URL 方案。我们强烈建议您为 iOS 和安卓平台使用相同的自定义 URL 方案。

</Callout>

要调取您的自定义 URL 方案，请按照下列步骤操作：

1. 使用 XCode 打开您的应用项目。
2. 从左侧菜单中选择您的项目。
3. 在 **Targets** 中选择您的应用。
4. 从顶部菜单中选择 **Info** 。
5. 展开 **URL Type** 部分，找到自定义 URL 方案。如果 URL 方案字段包含版本设置 \(如 `$(CUSTOM_URL_SCHEME)`\)，请前往版本设置界面，调取自定义 URL 方案值：
   1. 选择顶部菜单中的 **Build Settings** 。
   2. 找到与 URL 方案字段中名称相同的设置，调取发布和调试值。

如果您的 iOS 应用尚无自定义 URL 方案，请按照下列步骤进行设置：

1. 使用 XCode 打开您的应用项目。
2. 从左侧菜单中选择您的项目。
3. 在 **Targets** 中选择您的应用。
4. 从顶部菜单中选择 **Info** 。
5. 展开 **URL Types** 板块。
6. 选择添加选项，以添加新 URL 类别。
7. 输入下列信息，以创建 URL 方案：
   - **标识符** ： `$(PRODUCT_BUNDLE_IDENTIFIER)`
   - **URL 方案** \- 您的自定义 URL 方案。该方案必须是唯一的。请勿使用受保护的方案，如 `http` 或 `https`。 `mailto`
   - **角色** ：编辑员

该方案适用于您的生产版本 **和** 调试版本。

### 链接解析域 {#link-resolution-domains}

<Callout type="tip">

对于通过电邮、短信、二维码和短链接生成平台的深度链接，链接解析域是必需的。请咨询您的营销团队，决定应用是否需要[链接解析](https://help.adjust.com/zh/article/link-resolution)。

</Callout>

您的营销团队可能已经在邮件营销平台中使用链接解析域。请向营销团队索取并保存这些信息，以备今后使用。
