Adjustダッシュボードで[コールバックURLを登録](https://help.adjust.com/ja/article/recommended-placeholders-callbacks)した場合、SDKがセッションを計測するとAdjustはコールバックURLにGETリクエストを送信します。

## [セッションコールバックパラメーター](session-callback-parameters)

<Callout type="note">

Adjust SDKは、セッションコールバックパラメーターをイベントコールバックパラメーターとマージします。イベントコールバックパラメーターは、セッションコールバックパラメーターより優先されます。つまり、パラメーターキーをイベントとセッションの両方に追加した場合、SDKはイベントパラメーターを送信します。

</Callout>

コールバックパラメーターをサーバーに設定できます。イベントにパラメーターを設定すると、SDKはそれらをコールバックURLに追加します。この情報を使用して、自社のBIシステムでユーザーのアプリ内行動を分析することが可能です。

### [グローバルコールバックパラメーターを追加する](add-global-callback-parameters)

<CodeBlock title="メソッドシグネチャー">

```dart
static void addGlobalCallbackParameter(String key, String value)
```

</CodeBlock>

イベントにコールバックパラメーターを追加するには、`String` key\-value引数を指定して`addGlobalCallbackParameter`メソッドを呼び出してください。このメソッドを複回呼び出すことで、複数のパラメーターを追加することができます。

```dart
Adjust.addGlobalCallbackParameter('key', 'value');
```

### [グローバルコールバックパラメーターを削除する](remove-global-callback-parameters)

<CodeBlock title="メソッドシグネチャー">

```dart
static void removeGlobalCallbackParameter(String key)
```

</CodeBlock>

不要になった場合は、特定のセッションコールバックパラメーターを削除できます。これを行うには、パラメーター`key`を`removeGlobalCallbackParameter`メソッドにパスします。

```dart
Adjust.removeGlobalCallbackParameter('key');
```

### [グローバルコールバックパラメーターをリセットする](reset-global-callback-parameters)

<CodeBlock title="メソッドシグネチャー">

```dart
static void removeGlobalCallbackParameters()
```

</CodeBlock>

不要になった場合は、全てのセッションパラメーターを削除できます。これを行うには、`removeGlobalCallbackParameters`メソッドを呼び出します。

```dart
Adjust.removeGlobalCallbackParameters();
```

## [グローバルパートナーパラメーター](global-partner-parameters)

[パートナーパラメーター](https://help.adjust.com/ja/article/data-sharing-ad-network#map-parameters)を追加することで、ネットワークパートナーに追加情報を送信することができます。

Adjustは、設定した[外部パートナー](https://help.adjust.com/ja/article/integrated-partners)にパートナーパラメーターを送信します。この情報は、より詳細な分析やリターゲティング施策にご利用いただけます。パートナー用にパラメーターの設定が完了し、機能が有効化されると、Adjustのサーバーはデータ転送を開始します。

<Callout type="note">

パートナーパラメーターはデフォルトではローデータに表示されません。ご希望の場合は、プレースホルダー`{partner_parameters}`を追加することで、それらを単一の文字列として受信することができます。

</Callout>

### [グローバルパートナーパラメーターを追加する](add-session-partner-parameters)

<CodeBlock title="メソッドシグネチャー">

```dart
static void addGlobalPartnerParameter(String key, String value)
```

</CodeBlock>

セッションと一緒にパートナーパラメーターを送信するには、`String` key\-value引数を指定して`addGlobalPartnerParameter`メソッドを呼び出してください。このメソッドを複回呼び出すことで、複数のパラメーターを追加することができます。

```dart
Adjust.addGlobalPartnerParameter('key', 'value');
```

### [グローバルパートナーパラメーターを削除する](remove-session-partner-parameters)

```dart

```

</CodeBlock>

不要になった場合は、特定のセッションパートナーパラメーターを削除できます。これを行うには、パラメーターキーを`removeGlobalPartnerParameter`メソッドにパスします。

```dart

```

### [グローバルパートナーパラメーターをリセットする](reset-global-partner-parameters)

<CodeBlock title="メソッドシグネチャー">

```dart
static void removeGlobalPartnerParameters()
```

</CodeBlock>

不要になった場合は、全てのセッションパートナーパラメーターを削除できます。これを行うには、`removeGlobalPartnerParameters`メソッドを呼び出します。

## [ディレイスタート](delay-start)

<Callout type="note">

SDKの動作開始を遅らせることができるのは最長 **10秒間** です。

</Callout>

Adjust SDKは、アプリの起動直後に動作を開始します。起動時に取得できないデータをセッションパラメーターで送信したい場合は、SDKの動作開始を遅らせることができます。それを行うには、Configオブジェクトの[`delayStart`プロパティ](/ja/sdk/flutter/configuration#delay-the-start-of-the-sdk)で、ディレイタイムを秒単位で設定してください。

```dart
AdjustConfig adjustConfig = new AdjustConfig('{YourAppToken}', AdjustEnvironment.Sandbox);
//...
adjustConfig.delayStart(5.5);
//...
Adjust.start(adjustConfig);
```

この時間が経過すると、SDKは情報をAdjustサーバーに送信します。`sendFirstPackages`メソッドを呼び出すことで、タイムアウトになる前に情報を送信できます。

```dart
Adjust.sendFirstPackages();
```
