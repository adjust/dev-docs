Use these methods to update the Adjust SDK's configuration.

<Function>

## initSdk

Initializes the Adjust SDK with a set of customizable options.

<slot slot="signature">

import InitSdkSig from "@web-signatures/Adjust/initSdk.mdx"

<InitSdkSig />

</slot>

<slot slot="snippet">

import InitSdk from "@web-examples/Adjust/initSdk.mdx"

<InitSdk />

</slot>

<slot slot="parameters">

`InitOptions` (InitOptions)
: A set of initialization options that configure the SDK's behavior.

<Accordion title="InitOptions">

`appToken` (string)
: Your Adjust app token.

`environment` (string)
: The environment your app is running in. `sandbox` or `production`.

`defaultTracker` (string)
: An optional campaign token to track installs against by default.

`externalDeviceId` (string)
: A custom device identifier.

`eventDeduplicationListLimit` (int)
: The number of custom event IDs to store.

`customUrl` (string)
: An optional custom endpoint to send requests to.

`dataResidency` (string)
: The region you want to store data in.

`urlStrategy` (string)
: The region whose endpoints you want to prioritize sending data to.

`namespace` (string)
: An optional custom namespace for data storage.

`attributionCallback` (function)
: An optional callback method called when the user's attribution information changes.

`logLevel` (string)
: The verbosity of logging.

`logOutput` (string)
: An optional custom HTML selector to send log output to.

</Accordion>

</slot>

</Function>
