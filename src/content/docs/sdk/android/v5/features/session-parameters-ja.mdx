Adjustダッシュボードで[コールバックURLを登録](https://help.adjust.com/ja/article/recommended-placeholders-callbacks)した場合、SDKがセッションを計測すると、AdjustはコールバックURLにGETリクエストを送信します。

## [グローバルコールバックパラメーター](global-callback-parameters)

<Callout type="note">

Adjust SDKは、グローバルコールバックパラメーターをイベントコールバックパラメーターとマージします。イベントコールバックパラメーターは、グローバルコールバックパラメーターより優先されます。つまり、パラメーターキーをイベントとセッションの両方に追加した場合、SDKはイベントパラメーターを送信します。

</Callout>

コールバックパラメーターをサーバーに設定できます。イベントにパラメーターを設定すると、SDKはそれらをコールバックURLに追加します。この情報を使用して、自社のBIシステムでユーザーのアプリ内行動を分析することが可能です。

### [addGlobalCallbackParameters](add-global-callback-parameters)

<CodeBlock title="メソッドシグネチャー">

```java
public static void addGlobalCallbackParameter(String key, String value)
```

</CodeBlock>

イベントにコールバックパラメーターを追加するには、`String` key\-value引数を指定して`Adjust.addGlobalCallbackParameters`メソッドを呼び出してください。このメソッドを複回呼び出すことで、複数のパラメーターを追加することができます。

<Tabs>
<Tab title="Kotlin" sync="kotlin">

```kotlin
Adjust.addGlobalCallbackParameter("foo", "bar")
```

</Tab>
<Tab title="Java" sync="java" >

```java
Adjust.addGlobalCallbackParameter("foo", "bar");
```

</Tab>
</Tabs>

### [グローバルコールバックパラメーターを削除する](remove-global-callback-parameters)

<CodeBlock title="メソッドシグネチャー">

```java
public static void removeGlobalCallbackParameter(String key)
```

</CodeBlock>

不要になった場合は、特定のセッションコールバックパラメーターを削除できます。これを行うには、グローバルパラメーター`key`を`Adjust.removeGlobalCallbackParameter`メソッドにパスします。

<Tabs>
<Tab title="Kotlin" sync="kotlin">
<CodeBlock highlight="foo">

```kotlin
Adjust.removeGlobalCallbackParameter("foo")
```

</CodeBlock>
</Tab>
<Tab title="Java" sync="java" >
<CodeBlock highlight="foo">

```java
Adjust.removeGlobalCallbackParameter("foo");
```

</CodeBlock>
</Tab>
</Tabs>

## [グローバルパートナーパラメーター](global-partner-parameters)

[パートナーパラメーター](https://help.adjust.com/ja/article/data-sharing-ad-network#map-parameters)を追加することで、ネットワークパートナーに追加情報を送信することができます。

Adjustは、設定した[外部パートナー](https://help.adjust.com/ja/article/integrated-partners)にパートナーパラメーターを送信します。この情報は、より詳細な分析やリターゲティング施策にご利用いただけます。パートナー用にパラメーターの設定が完了し、機能が有効化されると、Adjustのサーバーはデータ転送を開始します。

<Callout type="note">

パートナーパラメーターはデフォルトではローデータに表示されません。ご希望の場合は、プレースホルダー`{partner_parameters}`を追加することで、それらを単一の文字列として受信することができます。

</Callout>

### [addGlobalPartnerParameters](add-global-partner-parameters)

<CodeBlock title="メソッドシグネチャー">

```java
public static void addGlobalPartnerParameter(String key, String value)
```

</CodeBlock>

セッションと一緒にパートナーパラメーターを送信するには、`String` key\-value引数を指定して`Adjust.addGlobalPartnerParameter`メソッドを呼び出してください。このメソッドを複回呼び出すことで、複数のパラメーターを追加することができます。

<Tabs>
<Tab title="Kotlin" sync="kotlin">

```kotlin
Adjust.addGlobalPartnerParameter("foo", "bar")
```

</Tab>
<Tab title="Java" sync="java" >

```java
Adjust.addGlobalPartnerParameter("foo", "bar");
```

</Tab>
<Tab title="Javascript" sync="js" >

```js
Adjust.addGlobalPartnerParameter("foo", "bar");
```

</Tab>
</Tabs>
