If you [register a callback URL](https://help.adjust.com/en/article/recommended-placeholders-callbacks) in the Adjust dashboard, Adjust sends a GET request to your callback URL when the SDK measures a session.

## Global callback parameters {#global-callback-parameters}

<Callout type="note">

The Adjust SDK merges global callback parameters with event callback parameters. Event callback parameters take priority over global callback parameters. This means that if you add a parameter key to both an event and a session, the SDK sends the event parameter.

</Callout>

You can configure callback parameters to your servers. Once you configure parameters on an event, the SDK appends them to your callback URL. You can use this information to analyze your users' in-app behavior with your BI system.

### Add global callback parameters {#add-global-callback-parameters}

<CodeBlock title="Method signature">

```java
public static void addGlobalCallbackParameter(String key, String value)
```

</CodeBlock>

Add callback parameters to your event by calling the `Adjust.addGlobalCallbackParameters` method with `String` key-value arguments. You can add multiple parameters by calling this method multiple times.

<Tabs>
<Tab title="Kotlin" sync="kotlin">

```kotlin
Adjust.addGlobalCallbackParameter("foo", "bar")
```

</Tab>

<Tab title="Java" sync="java" >

```java
Adjust.addGlobalCallbackParameter("foo", "bar");
```

</Tab>
</Tabs>

### Remove global callback parameters {#remove-global-callback-parameters}

<CodeBlock title="Method signature">

```java
public static void removeGlobalCallbackParameter(String key)
```

</CodeBlock>

You can remove specific global callback parameters if they're no longer required. To do this, pass the parameter `key` to the `Adjust.removeGlobalCallbackParameter` method.

<Tabs>
<Tab title="Kotlin" sync="kotlin">

<CodeBlock highlight="foo">

```kotlin
Adjust.removeGlobalCallbackParameter("foo")
```

</CodeBlock>

</Tab>
<Tab title="Java" sync="java" >

<CodeBlock highlight="foo">

```java

```

</CodeBlock>

</Tab>
</Tabs>

### Remove all global callback parameters {#remove-all-global-callback-parameters}

<CodeBlock title="Method signature">

```java
public static void removeGlobalCallbackParameters()
```

</CodeBlock>

You can remove all session parameters if they're no longer required. To do this, call the `Adjust.removeGlobalCallbackParameters` method.

<Tabs>

```kotlin

```

</Tab>

## Global partner parameters {#global-partner-parameters}

You can send extra information to your network partners by adding [partner parameters](https://help.adjust.com/en/article/data-sharing-ad-network#map-parameters).

Adjust sends partner parameters to [external partners](https://help.adjust.com/en/article/integrated-partners) you've set up. This information is useful for more granular analysis and retargeting purposes. Adjust's servers forward these parameters once you've set them up and enabled them for a partner.

<Callout type="note">

Partner parameters don't appear in raw data by default. You can add the `{partner_parameters}` placeholder to receive them as a single string.

## Delay start {#delay-start}

<Callout type="note">

You can delay the startup of the SDK by up to **10 seconds**.

</Callout>

The Adjust SDK starts as soon as your app opens. If you want to send data that's not available at launch in session parameters, you can delay the start of the SDK. To do this, pass the delay time in seconds to the [`setDelayStart` method](/en/sdk/android/configuration#delay-the-start-of-the-sdk) on your config object.

<Tabs>
<Tab title="Kotlin" sync="kotlin">

```kotlin
val config = AdjustConfig(this, appToken, environment)
config.setDelayStart(5.5)
Adjust.onCreate(config)
```

</Tab>
<Tab title="Java" sync="java">

```java
AdjustConfig config = new AdjustConfig(this, appToken, environment);
config.setDelayStart(5.5);
Adjust.onCreate(config);
```

</Tab>
<Tab title="Javascript" sync="js">

```js
var adjustConfig = new AdjustConfig(yourAppToken, environment);
adjustConfig.setDelayStart(5.5);
Adjust.onCreate(adjustConfig);
```

</Tab>
</Tabs>

After this time has elapsed, the SDK sends the information to Adjust's servers. You can send the information before the timeout by calling the `sendFirstPackages` method.

<Tabs>
<Tab title="Kotlin" sync="kotlin">

```kotlin
Adjust.sendFirstPackages()
```

</Tab>

<Tab title="Java" sync="java" >

```java
Adjust.sendFirstPackages();
```

</Tab>

<Tab title="Javascript" sync="js" >

```js
Adjust.sendFirstPackages();
```

</Tab>
</Tabs>
