Tables are used to group information together in a readable way. All tables in Adjust's documentation render to [Atlas TableV2 components](https://atlas.adeven.com/docs/components/TableV2).

There are two syntaxes for writing tables:

-  [Markdown syntax](#markdown-tables)
-  [List table syntax](#list-tables)

Markdown tables should be preferred for their readability in plain text. List tables offer more flexibility with list items.

## Props

All tables can be styled by including different `props` to control their behavior.

<Table>

| Prop        | Data type | Required | Description                                                                                    |
| ----------- | --------- | -------- | ---------------------------------------------------------------------------------------------- |
| `list`      | Boolean   | No       | Whether the table is a list table. Defaults to `false` (Markdown table).                       |
| `height`    | Number    | No       | The height (in pixels) that the table should take up.                                          |
| `search`    | Boolean   | No       | Whether to include a search input. Defaults to `false`.                                        |
| `resizable` | Boolean   | No       | Whether readers should be able to resize columns. If not set, columns are sized automatically. |
| `noborder`  | Boolean   | No       | Whether to remove the table's borders. Borders are added by default.                           |
| `noflex`    | Boolean   | No       | Whether to prevent the table from taking up the full width of its parent element (default).    |
| `striped`   | Boolean   | No       | Whether the table should be highlighted by alternating rows.                                   |

</Table>

## Markdown tables

Tables can be written as [Markdown tables](https://www.markdownguide.org/extended-syntax/#tables) and converted to Atlas tables using the `<Table>` component.

### Example

```mdx
<Table>

| Header 1                       | Header 2                                  | Header 3                                     |
| ------------------------------ | ----------------------------------------- | -------------------------------------------- |
| Column 1                       | Column 2                                  | Column 3                                     |
| **Inline** _Markdown_ ~syntax~ | [is available](https://markdownguide.org) | <ul><li>So</li><li>is</li><li>HTML</li></ul> |

</Table>
```

<Table>

| Header 1                       | Header 2                                  | Header 3                                     |
| ------------------------------ | ----------------------------------------- | -------------------------------------------- |
| Column 1                       | Column 2                                  | Column 3                                     |
| **Inline** _Markdown_ ~syntax~ | [is available](https://markdownguide.org) | <ul><li>So</li><li>is</li><li>HTML</li></ul> |

</Table>

#### Column alignment

Columns in Markdown tables can be aligned using the [Markdown alignment syntax](https://www.markdownguide.org/extended-syntax/#alignment). If no alignment directive is included, columns will align to the **left**.

```mdx title="Aligned columns" {4}
<Table>

| Left aligned | Center aligned | Right aligned |
| :----------- | :------------: | ------------: |
| Left         |     Center     |         Right |

</Table>
```

| :----------- | :------------: | ------------: |

</Table>

## List tables

List tables are a special table syntax that mimic Sphinx's [`list-table` syntax](https://docutils.sourceforge.io/docs/ref/rst/directives.html#list-table).

### Example

```mdx
<Table list>

-  -  Header 1
   -  Header 2
   -  Header 3
-  -  Column 1
   -  Column 2
   -  Column 3
-  -  **Markdown** _Syntax_
   -  [is available](https://markdownguide.org)
   -  -  Including
      -  List
      -  Items

</Table>
```

</Table>

## Table variants

Both **Markdown** tables and **List** tables can be styled by passing props. These properties alter the behavior and appearance of tables in different ways.

### Set height table

You can explicitly set the height of a table in pixels. This allows you to control how much space a table takes up and improves the performance of rendering large tables. Tables that exceed this size are **scrollable**.

```mdx collapse={6-15} 'height="400"'
<Table height="400">

| Parameter             | Format  | In    | Description                                                                                                                   |
| --------------------- | ------- | ----- | ----------------------------------------------------------------------------------------------------------------------------- |
| `app_tokens`          | String  | Query | A comma-separated list of app tokens.                                                                                         |
| `start_date`          | Date    | Query | The start date of the selected period. `YYYY-MM-DD` format.                                                                   |
| `end_date`            | Date    | Query | The end date of the selected period. `YYYY-MM-DD` format.                                                                     |
| `utc_offset`          | Time    | Query | UTC offset for timezones. `[+-]HH:MM` format.                                                                                 |
| `kpis`                | String  | Query | Comma-separated list of metrics. Comma-separated list of metrics. Use App metrics, Fraud metrics, and Ad spend metrics        |
| `sandbox`             | Boolean | Query | Whether results come from sandbox or production data. Defaults to production (`false`).                                       |
| `countries`           | String  | Query | Comma-separated list of ISO 3166 alpha-2 country names.                                                                       |
| `os_names`            | String  | Query | Comma-separated list of OS names.                                                                                             |
| `device_types`        | String  | Query | Comma-separated list of device types.                                                                                         |
| `grouping`            | String  | Query | Grouping parameters. [See Result grouping](/en/api/kpi-service/grouping) for more information.                                |
| `tracker_filter`      | String  | Query | The tracker tokens of any trackers you want to include in your results. Only metrics relating to these trackers are returned. |
| `human_readable_kpis` | Boolean | Query | Replace metrics with human readable alternatives (for example: "Lifetime Value" instead of `lifetime_value`).                 |

</Table>
```

### Resizable table

If a table is resizable, users can resize columns to make content more readable.

<Callout type="important">

Resizable tables can't flex to the width of their parent. They are treated as [Noflex tables](#noflex-tables).

</Callout>

```mdx title="Resizable table" "resizable"
<Table resizable>

| Header 1 | Header 2 | Header 3 |
| -------- | -------- | -------- |
| Column 1 | Column 2 | Column 3 |

</Table>
```

<Table resizable>

| Header 1 | Header 2 | Header 3 |
| -------- | -------- | -------- |
| Column 1 | Column 2 | Column 3 |

</Table>

### Noflex tables

If you don't want a table to take up the width of its parent, pass the `noflex` prop.

```mdx title="Noflex table" "noflex"
<Table noflex>

| Header 1 | Header 2 | Header 3 |
| -------- | -------- | -------- |
| Column 1 | Column 2 | Column 3 |

</Table>
```

<Table noflex>

| Header 1 | Header 2 | Header 3 |
| -------- | -------- | -------- |
| Column 1 | Column 2 | Column 3 |

</Table>

### Striped tables

Striped tables have each alternating row highlighted to break up the presentation

```mdx title="Striped table" "striped"
<Table striped>

| Header 1    | Header 2    | Header 3    |
| ----------- | ----------- | ----------- |
| Row 1 col 1 | Row 1 col 2 | Row 1 col 3 |
| Row 2 col 1 | Row 2 col 2 | Row 2 col 3 |

</Table>
```

| ----------- | ----------- | ----------- |
| Row 1 col 1 | Row 1 col 2 | Row 1 col 3 |
| Row 2 col 1 | Row 2 col 2 | Row 2 col 3 |

</Table>

### Searchable table

If a table is searchable, a text input appears above the table. All columns are filtered by the search input.

```mdx title="Searchable table" "search"
<Table search>

| Column 1  | Column 2                  | Column 3                   |
| --------- | ------------------------- | -------------------------- |
| Apple     | Apples are nice fruit     | You should search for them |
| Pear      | Pears are nice fruit      | You should search for them |
| Grape     | Grapes are nice fruit     | You should search for them |
| Banana    | Bananas are nice fruit    | You should search for them |
| Pineapple | Pineapples are nice fruit | You should search for them |

</Table>
```

### Borderedless tables

Tables have a border by default. If you want to remove it, pass the `noborder` prop.

```mdx title="Borderless table" "noborder"
<Table noborder>

| Header 1    | Header 2    | Header 3    |
| ----------- | ----------- | ----------- |
| Row 1 col 1 | Row 1 col 2 | Row 1 col 3 |
| Row 2 col 1 | Row 2 col 2 | Row 2 col 3 |

</Table>
```

| ----------- | ----------- | ----------- |
| Row 1 col 1 | Row 1 col 2 | Row 1 col 3 |
| Row 2 col 1 | Row 2 col 2 | Row 2 col 3 |

</Table>
