Adjust SDK의 동작을 설정하는 방법에 관한 본 문서의 메서드를 참조하시기 바랍니다.

config 객체 인스턴스화 \{\#instantiate\-your\-config\-object\}
-------------------------------------------------------------

<codeblock title="메서드 서명"> 

```ts
constructor(appToken: string, environment: AdjustEnvironment)
```

</CodeBlock> 

Adjust SDK를 설정하려면 `AdjustConfig` 객체를 인스턴스화해야 합니다. 이 객체에는 Adjust SDK로 전달하기 위해 필요한 **읽기 전용** 구성 옵션이 포함되어 있습니다.

config 객체를 인스턴스화 하려면 새로운 `AdjustConfig` 인스턴스를 생성하고 다음 파라미터를 전달합니다.

* `appToken` `string`: [Adjust 앱 토큰](https://help.adjust.com/en/article/app-token-and-reporting-currency#view-your-app-details)입니다.
* `environment` \(`AdjustEnvironment`\): SDK를 실행할 환경입니다. 테스트를 위해 샌드박스 모드에서 SDK를 실행하려면 `AdjustConfig.EnvironmentSandbox`를 전달합니다. 릴리즈를 위해 프로덕션 모드에서 SDK를 실행하려면 `AdjustConfig.EnvironmentProduction`을 전달합니다.

```js
var adjustConfig = new AdjustConfig(
   "{YourAppToken}",
   AdjustConfig.EnvironmentSandbox,
);
Adjust.initSdk(adjustConfig);
```

읽기 전용 구성 \{\#readonly\-configuration\}
------------------------------------------

**읽기 전용** 구성 옵션은 SDK를 초기화하기 **전** 에 `AdjustConfig` 인스턴스에 설정되어 있습니다. SDK 실행 중에는 이 옵션을 변경할 수 없습니다. 사용할 옵션은 `Adjust.initSdk`를 실행하기 전에 설정해야 합니다.

### 로그 수준 설정 \{\#set\-your\-logging\-level\}

<codeblock title="메서드 서명"> 

```ts
setLogLevel(logLevel: AdjustLogLevel): void
```

</CodeBlock> 

Adjust SDK는 다양한 양의 정보를 반환하기 위해 구성 가능한 로그 레벨을 제공합니다. 이용 가능한 로그 레벨은 다음과 같습니다.

|         logLevel          |           설명            |
|---------------------------|-------------------------|
| `AdjustLogLevel.Verbose`  | 모든 로깅 활성화               |
| `AdjustLogLevel.Debug`    | 디버그 로깅 활성화              |
| `AdjustLogLevel.Info`     | 정보 수준 로그만 표시\(기본 설정\) |
| `AdjustLogLevel.Warn`     | 정보 로깅 비활성화              |
| `AdjustLogLevel.Error`    | 경고 수준 로깅 이하 비활성화        |
| `AdjustLogLevel.Assert`   | 오류 수준 로깅 이하 비활성화        |
| `AdjustLogLevel.Suppress` | 모든 로깅 비활성화              |

다음 인수로 `setLogLevel` 메서드를 호출하여 로그 수준을 설정할 수 있습니다.

* `logLevel` \(`AdjustLogLevel`\): 사용할 로그 수준

```js
var adjustConfig = new AdjustConfig(
   "{YourAppToken}",
   AdjustConfig.EnvironmentSandbox,
);
adjustConfig.setLogLevel(AdjustLogLevel.Verbose);
Adjust.initSdk(adjustConfig);
```

### 외부 디바이스 ID 설정 \{\#set\-external\-device\-identifier\}

<codeblock title="메서드 서명"> 

```ts
setExternalDeviceId(externalDeviceId: string): void
```

</CodeBlock> 

외부 디바이스 ID는 기기 또는 사용자에 할당할 수 있는 커스텀 값입니다. 이를 통해 세션과 플랫폼에 걸쳐 사용자를 식별할 수 있습니다. 또한 사용자별 설치 중복 제거에 도움이 되어, 동일한 사용자가 여러 번의 신규 설치로 집계되지 않습니다. 시작하려면 Adjust 담당자에게 문의하시기 바랍니다.

외부 기기 ID를 기기에 대한 커스텀 ID로서 사용할 수도 있습니다. 이는 기타 시스템과 통일성을 유지하고 싶은 경우 유용합니다.  다음 인수로 `setExternalDeviceId` 메서드를 호출하여 외부 기기 ID를 할당합니다.

* `externalDeviceId` \(`string`\): 외부 기기 ID. 이 값은 **대소문자를 구분** 합니다. 외부 기기 ID를 불러오기한 경우, 전달한 값이 불러온 값과 일치하는지 확인하시기 바랍니다.

<callout type="seealso"> 

자세한 내용은 Adjust 헬프 센터의 [외부 디바이스 ID 문서](https://help.adjust.com/en/article/external-device-identifiers)를 참조하시기 바랍니다.
</Callout> 

```js
var adjustConfig = new AdjustConfig(
   "{YourAppToken}",
   AdjustConfig.EnvironmentSandbox,
);
adjustConfig.setExternalDeviceId("{Your-External-Device-Id}");
Adjust.initSdk(adjustConfig);
```

비즈니스 애널리틱스에서 외부 기기 ID를 사용하려면 이를 [글로벌 콜백 파라미터](/en/sdk/cordova/features/session-parameters)로 전송해야 합니다.

Adjust로 이미 존재하는 외부 기기 ID를 불러오기 하는것도 가능합니다. 이를 통해 Adjust 서버는 미래 데이터를 기존의 기기 기록과 매칭할 수 있습니다. 세부 정보는 Adjust 담당자에게 문의하시기 바랍니다.

### 디폴트 링크 토큰 설정 \{\#set\-default\-link\-token\}

<codeblock title="메서드 서명"> 

```ts
setDefaultTracker(defaultTracker: string): void
```

</CodeBlock> 

앱이 기기에 사전 설치된 경우 디폴트 링크 토큰을 설정할 수 있습니다. 사용자가 사전 설치된 앱을 처음으로 실행할 때 해당 설치가 디폴트 링크 토큰으로 어트리뷰션됩니다. 디폴트 링크 토큰을 설정하려면 다음 인수로 `setDefaultTracker` 메서드를 호출합니다.

* `defaultTracker` \(`string`\): 사전 설치된 설치 내용을 기록할 [Adjust 링크 토큰](https://help.adjust.com/en/article/links#adjust-link-token)

```js
var adjustConfig = new AdjustConfig(
   "{YourAppToken}",
   AdjustConfig.EnvironmentSandbox,
);
adjustConfig.setDefaultTracker("{LinkToken}");
Adjust.initSdk(adjustConfig);
```

### 어트리뷰션에서 광고 비용 데이터 수신 \{\#receive\-ad\-spend\-data\}

<codeblock title="메서드 서명"> 

```ts
enableCostDataInAttribution(): void
```

</CodeBlock> 

기본 설정에 따라 Adjust SDK는 사용자의 어트리뷰션에서 광고 비용 데이터를 수신하지 않습니다. `AdjustConfig` 인스턴스의 `enableCostDataInAttribution` 메서드를 호출하여 이 정보를 요청하도록 SDK를 구성할 수 있습니다.

광고 비용 데이터는 사용자의 [어트리뷰션 정보](/en/sdk/cordova/features/attribution)에서 액세스할 수 있습니다.

```js
var adjustConfig = new AdjustConfig(
   "{YourAppToken}",
   AdjustConfig.EnvironmentSandbox,
);
adjustConfig.enableCostDataInAttribution();
Adjust.initSdk(adjustConfig);
```

### 백그라운드에서 전송 활성화 \{\#enable\-background\-recording\}

<codeblock title="메서드 서명"> 

```ts
enableSendingInBackground(): void
```

</CodeBlock> 

기본값에 따라 Adjust SDK는 앱이 백그라운드에서 실행 중일 때 요청 전송을 일시 중지합니다. 백그라운드 리코딩을 활성화하여 백그라운드에서 요청을 보내도록 SDK를 구성할 수 있습니다. 백그라운드 기록을 활성화하려면 `AdjustConfig` 인스턴스의 `enableSendingInBackground` 메서드를 호출합니다.

```js
var adjustConfig = new AdjustConfig(
   "{YourAppToken}",
   AdjustConfig.EnvironmentSandbox,
);
adjustConfig.enableSendingInBackground();
Adjust.initSdk(adjustConfig);
```

동적 구성 \{\#dynamic\-configuration\}
--------------------------------------

**동적** 구성 옵션은 이벤트 또는 사용자 행동에 대한 응답으로 SDK의 라이프사이클 중에 변경될 수 있습니다.

### 오프라인 모드 활성화 \{\#activate\-offline\-mode\}

<codeblock title="메서드 서명"> 

```ts
switchToOfflineMode(): void
```

</CodeBlock> <callout type="important"> 

오프라인 모드 설정은 세션간에 유지되지 않습니다. 오프라인 모드는 새 세션이 시작될 때 비활성화됩니다.
</Callout> 

Adjust SDK는 실시간으로 이벤트와 세션 데이터를 Adjust 서버로 전송합니다. SDK를 오프라인 모드로 설정하면 정보 전송이 일시 중단됩니다. 오프라인 모드에서 SDK는 모든 데이터를 기기의 로컬 파일에 저장합니다. 이후 SDK의 오프라인 모드가 해제되면 Adjust 서버로 해당 정보가 전송됩니다.

`Adjust.switchToOfflineMode` 메서드를 호출하여 언제든지 오프라인 모드를 토글할 수 있습니다.

```js
Adjust.switchToOfflineMode();
```

### 오프라인 모드 비활성화 \{\#deactivate\-offline\-mode\}

<codeblock title="메서드 서명"> 

```ts
switchBackToOnlineMode(): void
```

</CodeBlock> 

`Adjust.switchBackToOnlineMode` 메서드를 호출하여 SDK를 재활성화할 수 있습니다. 이렇게 하면 SDK가 정보 전송을 재개합니다.

```js
Adjust.switchBackToOnlineMode();
```

### 푸시 토큰 설정 \{\#set\-push\-tokens\}

<codeblock title="메서드 서명"> 

```ts
setPushToken(pushToken: string): void
```

</CodeBlock> 

푸시 토큰은 [Audiences](https://help.adjust.com/en/article/audiences)와 클라이언트 콜백에 사용되고 [삭제 및 재설치 트래킹](https://help.adjust.com/en/article/uninstalls-reinstalls)에도 필요합니다.

config 객체에는 푸시 토큰을 저장할 수 있는 문자열 `pushToken` 속성이 포함되어 있습니다. `Adjust.setPushToken` 메서드를 호출하고 다음 인수를 전달하여 언제든지 이 속성을 업데이트할 수 있습니다.

* `token` \(`string`\): 푸시 토큰

```js
Adjust.setPushToken("{PushNotificationsToken}");
```

### SDK 비활성화 \{\#disable\-the\-sdk\}

<codeblock title="메서드 서명"> 

```ts
disable(): void
```

</CodeBlock> 

Adjust SDK는 기본 설정에 따라 앱이 열려 있을 때 실행됩니다. `Adjust.disable` 메서드를 호출하면 Adjust SDK가 Adjut로 정보를 전송하는 것을 중지시킬 수 있습니다. Adjust SDK를 비활성화하면 Adjust로 데이터가 전송되지 않으며, SDK에 어떠한 정보도 기록되지 않습니다. 즉, SDK가 비활성화되었을 때 호출된 Adjust 메서드는 아무것도 기록하지 않습니다.

```js
Adjust.disable();
```

### SDK 활성화 \{\#enable\-the\-sdk\}

<codeblock title="메서드 서명"> 

```ts
enable(): void
```

</CodeBlock> 

SDK를 비활성화한 후에 다시 활성화하려면 `Adjust.enable` 메서드를 호출합니다. SDK가 활성화되면 Adjust 서버로 정보를 전송합니다.

```js
Adjust.enable();
```

#### 활성화 상태 조회 \{\#check\-enabled\-status\}

<codeblock title="메서드 서명"> 

```ts
isEnabled(): Promise<boolean id="sl-md0000000">
```

</CodeBlock> 

언제든지 콜백 함수와 함께 `Adjust.isEnabled` 메서드를 호출하여 Adjust SDK가 활성화되어 있는지 확인할 수 있습니다. 상태는 콜백 함수에 `boolean` 값으로 반환되며, 이는 SDK의 **활성화** \(`true`\) 또는 **비활성화** \(`false`\) 여부를 나타냅니다.

```js
Adjust.isEnabled(function (isEnabled) {
   if (isEnabled) {
      console.log("SDK is enabled");
   } else {
      console.log("SDK is disabled");
   }
});
```
