`/status`エンドポイントを使用して、新規アプリのステータスを確認できます。このエンドポイントは、`_Template`アプリの設定を新規アプリにコピーするプロセスのステータスを返します。
<codeblock title="エンドポイント" highlight="/status, {ticket_token}"> 

```http
GET https://settings.adjust.com/api/status/{ticket_token}
```

</CodeBlock> 

コピープロセスを取得する \{\#fetch\-copy\-process\}
--------------------------------------------

このエンドポイントを使用すると、設定コピー処理のステータスを返すことができます。`ticket_token` は、[`/app` エンドポイント](/en/api/app-automation-api/app-endpoint) を使用してアプリを作成した際に取得されます。

|        名前         |  データタイプ  | 入力する値 |                          説明                           |
|-------------------|----------|-------|-------------------------------------------------------|
| `ticket_token`\* | `String` | パス    | 確認したいチケットのトークン。<br />`/app`エンドポイントはアプリを作成する際にこれを返します。 |

```json
{
   "status": "String"
}
```

### ステータス \{\#statuses\}

* `NEW`: チケットが作成されたものの、移行ジョブが開始していません。
* `OPENED`: このチケットのジョブが開始しました。
* `FAILED`: ジョブが失敗しました。
* `COMPLETED`: ジョブが正常に完了しました。

例 \{\#example\}
------------------

<codeblock title="cURL" highlight="<ticket_token>"> 

```console
$ curl \
--header "AdjustAuthorization: Token <adjust api token id="sl-md0000000">" \
--header "SignatureAuthorization: Token <adjust signature token id="sl-md0000000">" \
-X GET https://settings.adjust.com/api/status/<ticket_token id="sl-md0000000">
```

</CodeBlock> <codeblock title="成功レスポンス">

</CodeBlock>
