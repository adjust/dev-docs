디바이스 API를 통해 사용자의 디바이스 정보를 확인하고, 테스트 목적으로 진행한 Adjust에서 디바이스 활동 내역을 삭제해보시기 바랍니다.

## [작동 방식](how-it-works)

1. **콜 인증**

   디바이스 API 엔드포인트를 사용하려면 Adjust API 토큰이 필요합니다. [인증 가이드](#authentication) 내 설명을 참조하여 해당 토큰을 API와 함께 사용하는 방법을 확인하시기 바랍니다.

2. **디바이스 세부사항 검사**

   [디바이스 엔드포인트 검사\(inspect device endpoint\)](#inspect-device)를 사용하여 기기 및 사용자 활동에 대한 세부 정보를 반환합니다.

3. **디바이스 삭제**

   Adjust에서의 디바이스 내역을 삭제하려면 [Forget device 엔드포인트](#forget-device)를 사용하시기 바랍니다. 이는 동일한 장치에서 발생한 다수의 설치를 테스트할 때 유용합니다.

## [인증](authentication)

디바이스 API를 이용하려면 [베어러 토큰](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication)이 필요합니다. 이 글의 단계에 따라 Adjust 대시보드에서 이 토큰을 받을 수 있습니다.

### [Adjust API 토큰의 위치](find-your-adjust-api-token)

API 콜을 시작하기 전에 우선 Adjust API 토큰을 찾아야 합니다. Adjust API 토큰은 Adjust 대시보드에서 찾을 수 있습니다.

<Callout type="important">

SSO를 설정한 경우 API 토큰은 Adjust 대시보드에서 확인할 수 없습니다. API 토큰 관련 지원이 필요하시다면 Adjust 담당자 또는 [support@adjust.com](mailto:support@adjust.com) 로 연락 바랍니다.

</Callout>

Adjust API 토큰은 **프로필** 탭에서 확인할 수 있습니다. Adjust API 토큰을 확인하려면 다음 단계를 따르시기 바랍니다.

1. 좌측 하단의 설정 아이콘을 선택합니다.
2. **계정 설정** 을 선택합니다. 계정 설정 페이지가 열립니다.
3. **내 프로필** 탭을 선택합니다.
4. **API 토큰** 은 **사용자 세부 사항** 과 함께 표시됩니다. 시스템 클립보드에 토큰을 복사하기 위해 복사를 선택합니다.

### [Adjust API 토큰 재설정](reset-your-adjust-api-token)

Adjust API 토큰은 언제든지 재설정할 수 있습니다. 토큰을 재설정하면 이전의 토큰은 무효화됩니다. 따라서 이전 토큰을 사용하는 모든 요청은 작동하지 않습니다. Adjust API 토큰을 재설정한 이후 토큰을 사용하는 모든 요청을 업데이트하시기 바랍니다.

1. 좌측 하단의 설정 아이콘을 선택합니다.
2. **계정 설정** 을 선택합니다. 계정 설정 페이지가 열립니다.
3. **내 프로필** 탭을 선택합니다.
4. **API 토큰** 은 **사용자 세부 사항** 과 함께 표시됩니다.
5. **API 토큰 재설정** 을 선택합니다. 팝업 모달창이 표시됩니다.
6. 모달에 계정 비밀번호를 입력한 후 API 토큰 재설정을 위해 **재설정** 을 선택합니다. 모달이 닫히고 화면 오른쪽 상단에 확인이 나타납니다.
7. 시스템 클립보드에 **API 토큰** 을 복사하기 위해 복사를 선택합니다.

## [기기 정보 불러오기](fetch-device-information)

기기 정보를 JSON 객체로 반환.

<Table>

| 파라미터           | 데이터 유형 | In    | 설명                                                          |
| ------------------ | ----------- | ----- | ------------------------------------------------------------- |
| `app_token`\*      | `String`    | Query | 앱의 12자리 ID                                                |
| `advertising_id`\* | `String`    | Query | 기기의 고유 광고 ID. 아래의 [광고 ID](#advertising-ids) 참조. |

<Accordion title="광고 ID(Advertising IDs)">

-  `idfa` \(iOS 기기\)
-  `gps_adid` \(Google 광고 ID\)
-  `android_id` \(Android ID\)
-  `fire_adid` \(Amazon Fire ID\)
-  `win_adid` \(Windows 광고 ID\)

</Accordion>

## [Forget device 엔드포인트](forget-device)

Adjust의 Forget device 엔드포인트를 통해 Adjust의 디바이스 활동 내역을 삭제하여 동일 디바이스에 여러 번의 설치 테스트를 시행해보시기 바랍니다.

<Callout type="seealso" title="시작에 앞서">

디바이스 삭제를 위해서는 **관리자** 또는 **편집자** 권한이 필요합니다. **커스텀 편집자** 권한을 가진 경우 대상 앱에 대한 편집 권한이 있어야 합니다. 자세한 정보는 [사용자 권한](https://help.adjust.com/ko/article/users) 페이지를 참고하시기 바랍니다.

</Callout>
<CodeBlock title="엔드포인트" highlight="/forget_device">

```http
POST https://api.adjust.com/device_service/api/v1/forget_device
```

</CodeBlock>

## [디바이스 삭제](forget-a-device)

Adjust에서 기기 정보 삭제

<Table>

| 파라미터      | 데이터 유형 | In     | 설명             |
| ------------- | ----------- | ------ | ---------------- |
| `app_token`\* | `String`    | 데이터 | 앱의 12자리 ID   |
| `adid`\*      | `String`    | 데이터 | 기기의 Adjust ID |

</Table>

## [예시](forget-device-example)

<CodeBlock title="cURL" highlight="gwzpeepw8uf8, acf8534f2f052395e617a38730682ccc">

```console
$ curl \
--header "Authorization: Token token=ask43jskdp2tg2hg87" \
--location --request POST "https://api.adjust.com/device_service/api/v1/forget_device" \
--data "adid=acf8534f2f052395e617a38730682ccc&app_token=gwzpeepw8uf8"
```

</CodeBlock>
<CodeBlock title="성공적인 응답">

```http
200: Forgot device
```

</CodeBlock>
