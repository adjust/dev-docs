하위 트래커 불러오기
-----------

앱에 해당하는 하위 트래커 리스트 반환.
<apiversion version="v2"> <table> 

|        파라미터        |  데이터 유형   |  In   |                  설명                  |
|--------------------|-----------|-------|--------------------------------------|
| `app_token`\*     | `String`  | Path  | 앱의 12자리 ID                           |
| `tracker_token`\* | `String`  | Path  | 상위 트래커 토큰\(parent tracker token\). |
| `cursor`           | `String`  | Query | 결과 컬렉션을 포함한 고유한 커서 위치.               |
| `limit`            | `Integer` | Query | 응답에 포함된 요소의 최대 갯수. 양의 정수 값이어야 합니다.   |

</Table> </ApiVersion> <apiversion version="v1"> <table> 

|        파라미터        |  데이터 유형   |  In   |                  설명                  |
|--------------------|-----------|-------|--------------------------------------|
| `app_token`\*     | `String`  | Path  | 앱의 12자리 ID                           |
| `tracker_token`\* | `String`  | Path  | 상위 트래커 토큰\(parent tracker token\). |
| `after`            | `String`  | Query | 다음 페이지의 요소를 가져오는 커서.                 |
| `before`           | `String`  | Query | 이전 페이지의 요소를 가져오는 커서.                 |
| `limit`            | `Integer` | Query | 응답에 포함된 요소의 최대 갯수. 양의 정수 값이어야 합니다.   |

</Table> </ApiVersion> 

### 페이지 구분

이 엔드포인트로부터의 결과는 커서를 사용하여 페이징됩니다. 각 페이지는 이전 및 다음 페이지의 위치를 반환합니다. 이를 콜에 포함하여 특정 페이지를 반환할 수 있습니다. 또한 한 페이지에 포함되는 아이템의 수를 제한할 수 있습니다. 기본값 제한은 `50`개 입니다.

각 결과 페이지는 커서 위치가 포함된 "paging" 객체를 포함합니다. 이 커서 위치를 사용하여 페이지를 이동할 수 있습니다. 예를 들어, 첫 페이지에는 다음과 같은 결과가 포함됩니다.
<apiversion version="v2"> 

```json
"paging": {
   "next": "https://api.adjust.com/public/v2/apps/gwzpeepw8uf8/trackers/abc123/children?cursor=g2wAAAACYhW1_gxkAANuaWxq",
   "cursor": "g2wAAAACYhW1_gxkAANuaWxq"
}
```

두 번째 페이지로 이동하려면 `next` 링크를 클릭하여 다음 지정된 `cursor` 위치로부터의 결과를 반환합니다.

```console
$ curl "https://api.adjust.com/public/v2/apps/gwzpeepw8uf8/trackers/abc123/children?cursor=g2wAAAACYhW1_gxkAANuaWxq"
```

</ApiVersion> <apiversion version="v1"> 

```json
"paging": {
   "page_size": "50",
   "collection_size": "49",
   "total": "199",
   "cursors": {
      "after": "g2wAAAACYhW1_gxkAANuaWxq",
      "before": null
   },
   "next": "https://api.adjust.com/public/v1/apps/gwzpeepw8uf8/trackers/abc123/children?after=g2wAAAACYhW1_gxkAANuaWxq&limit=50",
   "previous": null
}
```

두 번째 페이지로 이동하려면 `after` 파라미터의 다음 요청에 `after` 값을 추가합니다. 예시:

```console
$ curl "https://api.adjust.com/public/v1/apps/gwzpeepw8uf8/trackers/abc123/children?after=g2wAAAACYhW1_gxkAANuaWxq"
```

</ApiVersion> 

### 응답 파라미터

<table search> 

|         필드          |  데이터 유형   |                                                           설명                                                           |                     예시                      |
|---------------------|-----------|------------------------------------------------------------------------------------------------------------------------|---------------------------------------------|
| `name`              | `String`  | 트래커 이름                                                                                                                 | `Adroll`                                    |
| `label`             | `String`  | 트래커 이름의 마지막 레벨 부분                                                                                                      | `Adroll`                                    |
| `level`             | `Integer` | 트래커의 레벨. 트래커의 레벨.<br/><ul><li>네트워크 = `1`</li><li>Campaign = `2`</li><li>Adgroup = `3`</li><li>Creative = `4`</li></ul> | `1`                                         |
| `archived`          | `Boolean` | 트래커의 아카이브 여부.                                                                                                          | `true`                                      |
| `has_subtrackers`   | `Boolean` | 트래커의 하위 트래커 포함 여부.                                                                                                     | `true`                                      |
| `partner_id`        | `Integer` | 트래커에 포함될 파트너 ID.                                                                                                       | `3`                                         |
| `cost_data_enabled` | `Boolean` | 트래커의 cost\_data 활성화 여부.                                                                                               | `false`                                     |
| `impression_url`    | `String`  | 노출 트래킹에 대한 트래커 URL.                                                                                                    | `https://view.adjust.com/impression/abc123` |
| `url`               | `String`  | 트래킹에 대한 트래커 URL.                                                                                                       | `https://app.adjust.com/abc123`             |
| `click_url`         | `String`  | 클릭 트래킹에 대한 트래커 URL.                                                                                                    | `https://app.adjust.com/abc123`             |

</Table>
