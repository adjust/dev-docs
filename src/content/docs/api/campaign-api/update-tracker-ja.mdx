このエンドポイントを使用して、アプリの既存のトラッカーまたはサブトラッカーを更新します。トラッカーに新規パートナーを追加したり、コストデータを有効にするかどうかを管理することができます。

```http
PATCH https://api.adjust.com/public/v1/apps/{app_token}/trackers/{tracker_token}
```

## [トラッカーを更新する](update-a-tracker)

アプリに関連するトラッカーまたはサブトラッカーを更新します。

### [パラメーター](parameters)

<Table>

| パラメーター        | データタイプ | 入力する値 | 説明                                                                                                                 |
| ------------------- | ------------ | ---------- | -------------------------------------------------------------------------------------------------------------------- |
| `app_token`\*       | `String`     | パス       | アプリの12桁の識別子。                                                                                               |
| `tracker_token`\*   | `String`     | パス       | 更新したいトラッカーのトークン。                                                                                     |
| `partner_id`        | `Integer`    | ボディ     | トラッカーに付与したいパートナーのID。partnersエンドポイントを使用して、パートナーとパートナーIDの一覧を取得します。 |
| `cost_data_enabled` | `Boolean`    | ボディ     | トラッカーのコストデータを有効化するかどうか。                                                                       |

</Table>

### [レスポンスパラメーター](response-parameters)

<Table search>

| フィールド          | データタイプ | 説明                                                                                  | 例                                          |
| ------------------- | ------------ | ------------------------------------------------------------------------------------- | ------------------------------------------- |
| `name`              | `String`     | トラッカーの名前。                                                                    | `Adroll`                                    |
| `label`             | `String`     | トラッカー名の最後のレベルの部分。                                                    | `Adroll`                                    |
| `level`             | `Integer`    | トラッカーのレベル。以下の\[トラッカーレベル\]\(tracker\-levels\)を参照してください。 | `1`                                         |
| `archived`          | `Boolean`    | トラッカーがアーカイブされたかどうか。                                                | `true`                                      |
| `has_subtrackers`   | `Boolean`    | トラッカーにサブトラッカーがあるかどうか。                                            | `true`                                      |
| `partner_id`        | `Integer`    | トラッカーに付与されるパートナーのID。                                                | `3`                                         |
| `cost_data_enabled` | `Boolean`    | トラッカーでcost_dataが有効になっているかどうか。                                     | `false`                                     |
| `impression_url`    | `String`     | インプレッション計測用のトラッカーのURL。                                             | `https://view.adjust.com/impression/abc123` |
| `url`               | `String`     | 計測用のトラッカーのURL。                                                             | `https://app.adjust.com/abc123`             |
| `click_url`         | `String`     | クリック計測用のトラッカーのURL。                                                     | `https://app.adjust.com/abc123`             |

</Table>
<Accordion title="トラッカーレベル">

-  ネットワーク = `1`
-  キャンペーン = `2`
-  アドグループ = `3`
-  クリエイティブ = `4`

</Accordion>

```json
{
   "name": "String",
   "token": "String",
   "label": "String",
   "level": 1,
   "archived": false,
   "has_subtrackers": false,
   "partner_id": 3,
   "cost_data_enabled": false,
   "url": "String",
   "click_url": "String",
   "impression_url": "String"
}
```

## [例](example)

```console
$ curl \
--header 'Authorization: Token token={API_TOKEN}' \
--header 'Content-Type: application/json' \
--location --request PATCH 'https://api.adjust.com/public/v1/apps/gwzpeepw8uf8/trackers/klm789' \
--data-raw '{"partner_id": 174, "cost_data_enabled": false}'
```
