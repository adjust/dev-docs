Adjust 提供 API 终端来完成行使被遗忘权 \(Right to be Forgotten，简称 RTBF\) 的请求。该终端会为给定设备完成下列操作：

-  从 Adjust 中永久删除指定应用中该用户的个人数据。
-  不再从指定应用的 Adjust SDK 接收该用户的信息。
-  从 Adjust 控制面板中移除指定应用中该用户数据的所有引用。
-  设置您的服务器，以便在用户发送 RTBF 请求时向此 API 终端发送请求。要进一步了解 RTBF 请求，请参考 Adjust 的[GDPR 相关文档](https://help.adjust.com/zh/article/gdpr)。

<Callout type="important">

此操作具有永久性。RTBF 请求发送后，Adjust 将无法恢复与此设备关联的数据，也无法在未来接收该设备的请求。

</Callout>

```http
POST https://gdpr.adjust.com/gdpr_forget_device
```

## 发送 RTBF 请求

发送包含应用识别码和设备 ID 的 POST 请求，移除于设备关联的所有数据。

<Callout type="note">

RTBF 请求仅适用于您在调用中指定的应用。其他应用的设备数据不会受影响。

</Callout>

<Table>

| 参数             | 数据类型 | 位置 | 描述                                                                     |
| ---------------- | -------- | ---- | ------------------------------------------------------------------------ |
| `app_token`\*    | 字符串   | 查询 | 控制面板上的 Adjust 应用识别码                                           |
| `s2s`\*          | 布尔     | 查询 | 调用是否通过服务器到服务器方式发送 \(S2S 调用\)。值必须为 `1` \(true\)   |
| `device_token`\* | 字符串   | 查询 | 需要 Adjust 遗忘的设备 ID。请查看[下方表格](#device-ids)，了解可用的值。 |

</Table>

### 设备标识符

请务必在请求中包含设备标识符。可用的标识符如下：

<Table>

| 参数        | 描述                        | 示例                                   |
| ----------- | --------------------------- | -------------------------------------- |
| `idfa`      | 适用于广告主的 iOS ID       | `8C6CBCOD-5F43-4765-A6E6-84DFF3D24707` |
| `idfv`      | 适用于供应商的 iOS ID       | `CCB300A0-DE1B-4D48-BC7E-599E453B8DD4` |
| `gps_adid`  | Google Play 应用商店广告 ID | `38400000-8cf0-11bd-b23e-10b96e40000d` |
| `fire_adid` | Fire OS 广告 ID             | `63c5519b-7e66-7ee6-aa5d-3b290743811f` |
| `win_adid`  | Windows 广告 ID             | `107e8ea14329d4a2194ebbb6dc0c0fd7`     |
| `adid`      | Adjust 设备 ID              | `18546f6171f67e29d1cb983322ad1329`     |

</Table>

## 示例

```console
$ curl -X POST "https://gdpr.adjust.com/gdpr_forget_device?s2s=1&app_token=kwrqhymtnsr2&idfa=9C5CBC1D-4F42-4764-A5E6-84DAF3D24707"
```

### 响应

<Table>

| 状态代码 | 含义               | 描述                                                                                                                                                  |
| -------- | ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| 200      | 好                 | 已收到并处理请求                                                                                                                                      |
| 400      | Bad request        | 格式或识别码无效。请仔细检查，确保在请求中包含了必要的参数。如需帮助，请联系您的账户经理或发送邮件至[support@adjust.com](mailto:support@adjust.com)。 |
| 429      | Too many requests  | 该终端的限制为每分钟 10 次请求，一旦超过此限制，则停止响应避免滥用。请晚些再次尝试发送请求。                                                          |
| 451      | 由于法规原因不可用 |                                                                                                                                                       |

</Table>
