Adjust SDK 이벤트 전송을 중지하고, 대신 S2S 이벤트를 전송하도록 설정을 변경할 수 있습니다.

S2S 설정으로 전환하는 경우 철저한 도입 계획을 세워야 합니다. 이를 통해 SDK와 서버로부터 중복 이벤트를 전송하는 것을 예방할 수 있습니다. 아래의 단계를 통해 전환을 완료하고, S2S 도입을 테스트 및 검증하시기 바랍니다.

[설정 단계](setup-steps)
--------------------

Adjust SDK 이벤트 전송에서 S2S 이벤트 전송으로 전환하려면 다음의 단계를 수행하시기 바랍니다.

1. 기존의 SDK 이벤트 토큰에 상응하는 새로운 S2S 이벤트 토큰을 생성합니다. Adjust는 새로운 이벤트 토큰에 명명 규칙을 적용할 것을 권장합니다. 
   * **예:** S2S 이벤트 이름을 `s2s_`로 시작하고, 이후에 상응하는 SDK 이벤트 이름이 오게 합니다.

2. S2S 이벤트 요청 URL에 필수 파라미터 및 추가하고 싶은 파라미터가 모두 포함되었는지 확인합니다. 
   * [파라미터 전체 목록 보기](https://help.adjust.com/en/article/server-to-server-events#required-parameters)

3. SDK 이벤트에 콜백 파라미터나 파트너 파라미터가 추가된 경우, 동일한 파라미터 이름과 값이 이에 상응하는 S2S 이벤트에 설정되도록 하시기 바랍니다. 
   * [iOS](/en/sdk/ios/features/events#add-callback-parameters)
   * [Android](/en/sdk/android/features/events#add-callback-parameters)

4. S2S 이벤트 콜백 설정
5. 서버에 S2S 이벤트를 도입합니다.
6. 이전의 SDK 이벤트 개수와 이에 상응하는 BI 시스템 내 신규 S2S 이벤트 개수를 비교합니다. 
   * 숫자가 일치하지 않는 경우, S2S 이벤트 로직을 수정하고 해결되지 않은 에러 메시지가 있는지 확인합니다.
   * 숫자가 일치할 때까지 5단계를 반복합니다.

7. SDK 이벤트 대신 S2S 이벤트에 기반하여 리포팅을 사용할 수 있도록 BI 시스템을 업데이트합니다.
8. 파트너로부터 SDK 이벤트의 연결을 해제합니다.
9. SDK 이벤트가 사용한 것과 동일한 이벤트 이름으로 파트너에 S2S 이벤트를 연결합니다.
10. 모든 파트너가 새로운 이벤트를 수신하는지 확인합니다.
11. SDK 이벤트를 삭제한 앱 업데이트를 출시합니다.
12. Adjust 대시보드에서 SDK 이벤트 토큰을 숨깁니다.
