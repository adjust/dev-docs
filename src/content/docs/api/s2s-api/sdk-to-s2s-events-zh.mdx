您可以变更设置，停止发送 Adjust SDK 事件，转而发送服务器到服务器 \(S2S\) 事件。

在向 S2S 事件转换时，您必须制定明确的计划并严格执行，这一点非常重要，这样才能避免 SDK 和服务器重复发送事件。请采用下方步骤来实施、测试并验证您的设置。

[设置步骤](setup-steps)
-------------------

要停止发送 Adjust SDK 事件，转而发送 S2S 事件，请按照下列步骤操作。

1. 创建与现有 SDK 事件识别码对应的新 S2S 事件识别码。我们推荐为新的事件识别码设置命名约定。 
   * **示例：** S2S 事件名称以`s2s_` 开头，之后为对应的 SDK 事件名称。

2. 检查并确保 S2S 事件请求 URL 带有所有必需参数，以及您需要的任何额外参数。 
   * [查看完整参数列表](https://help.adjust.com/en/article/server-to-server-events#required-parameters)。

3. 如果 SDK 事件附加有回传参数或合作伙伴参数，请确保与之对应的 S2S 事件带有名称和值均相同的参数。 
   * [iOS](/en/sdk/ios/features/events#add-callback-parameters)
   * [安卓](/en/sdk/android/features/events#add-callback-parameters)

4. 为 S2S 事件设置回传。
5. 在服务器上部署 S2S 事件。
6. 在您的 BI 系统中比较旧 SDK 事件与对应新 S2S 事件的数量。 
   * 如果新旧事件数量不匹配，请修复 S2S 事件逻辑，并仔细检查是否有尚未解决的错误信息。
   * 重复步骤 5，直至数量相符。

7. 更新 BI 系统，使其使用基于 S2S 事件的报告，而非基于 SDK 事件。
8. 解除 SDK 事件与合作伙伴的关联。
9. 使用与 SDK 事件相同的事件名称，将 S2S 事件关联到合作伙伴。
10. 验证并确保所有合作伙伴都能收到新事件。
11. 发布已移除 SDK 事件的应用更新。
12. 在 Adjust 控制面板中隐藏 SDK 事件识别码。
