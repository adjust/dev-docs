캠페인 API를 사용하면 링크, 하위 링크 및 파트너 정보를 얻을 수 있습니다. 새 링크를 생성하고 기존 링크를 새 파트너 정보로 업데이트하시기 바랍니다.

## [작동 방식](how-it-works)

1. **콜 인증**

   캠페인 API 엔드포인트를 사용하려면 Adjust API 토큰이 필요합니다. [인증 가이드](#authentication) 내 설명을 참조하여 해당 토큰을 API와 함께 사용하는 방법을 확인하시기 바랍니다.

2. **앱과 관련된 네트워크 링크 목록 수신**

   `trackers` 엔드포인트를 사용하여 앱과 관련된 모든 링크에 대한 세부 정보를 가져옵니다. [최상위 링크](#fetch-links) 또는 [하위 링크](#fetch-sublinks) 목록을 가져올 수 있습니다.

3. **새 네트워크 링크 생성**

   [`trackers` 엔드포인트에 POST 요청](#create-links)을 전송하여 앱에 새 링크를 추가할 수 있습니다. 이 엔드포인트는 최상위 링크와 하위 링크 생성에 사용됩니다.

4. **파트너 목록 수신**

   [`partners` 엔드포인트](#fetch-partners)를 사용하면 파트너 목록을 받을 수 있으며, 링크 업데이트 시 활용할 수 있습니다.

5. **새 파트너로 링크 업데이트**

   [`trackers` 엔드포인트에 PATCH 요청](#update-links)을 전송하여 기존 링크를 새 파트너와 비용 데이터 정보로 업데이트할 수 있습니다.

## [인증](authentication)

캠페인 API를 이용하려면 [베어러 토큰](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication)이 필요합니다. 아래의 단계에 따라 Adjust 대시보드에서 이 토큰을 받을 수 있습니다.

### [Adjust API 토큰의 위치](find-your-adjust-api-token)

API 콜을 시작하기 전에 우선 Adjust API 토큰을 찾아야 합니다. Adjust API 토큰은 Adjust 대시보드에서 찾을 수 있습니다.

<Callout type="important">

SSO를 설정한 경우 API 토큰은 Adjust 대시보드에서 확인할 수 없습니다. API 토큰 관련 지원이 필요하시다면 Adjust 담당자 또는 [support@adjust.com](mailto:support@adjust.com) 로 연락 바랍니다.

</Callout>

Adjust API 토큰은 **프로필** 탭에서 확인할 수 있습니다. Adjust API 토큰을 확인하려면 다음 단계를 따르시기 바랍니다.

1. 좌측 하단의 설정 아이콘을 선택합니다.
2. **계정 설정** 을 선택합니다. 계정 설정 페이지가 열립니다.
3. **내 프로필** 탭을 선택합니다.
4. **API 토큰** 은 **사용자 세부 사항** 과 함께 표시됩니다. 시스템 클립보드에 토큰을 복사하기 위해 복사를 선택합니다.

### [Adjust API 토큰 재설정](reset-your-adjust-api-token)

Adjust API 토큰은 언제든지 재설정할 수 있습니다. 토큰을 재설정하면 이전의 토큰은 무효화됩니다. 따라서 이전 토큰을 사용하는 모든 요청은 작동하지 않습니다. Adjust API 토큰을 재설정한 이후 토큰을 사용하는 모든 요청을 업데이트하시기 바랍니다.

1. 좌측 하단의 설정 아이콘을 선택합니다.
2. **계정 설정** 을 선택합니다. 계정 설정 페이지가 열립니다.
3. **내 프로필** 탭을 선택합니다.
4. **API 토큰** 은 **사용자 세부 사항** 과 함께 표시됩니다.
5. **API 토큰 재설정** 을 선택합니다. 팝업 모달창이 표시됩니다.
6. 모달에 계정 비밀번호를 입력한 후 API 토큰 재설정을 위해 **재설정** 을 선택합니다. 모달이 닫히고 화면 오른쪽 상단에 확인이 나타납니다.
7. 시스템 클립보드에 **API 토큰** 을 복사하기 위해 복사를 선택합니다.

## [파트너 불러오기](fetch-partners)

<CodeBlock title="엔드포인트" highlight="/partners">

```http
GET https://api.adjust.com/public/v1/partners
```

</CodeBlock>

본 엔드포인트를 사용하여 파트너 및 관련 데이터 목록을 불러오시기 바랍니다.

### [파라미터](fetch-partners-parameters)

<Table>

| 파라미터 | 데이터 유형 | In    | 설명                                                       |
| -------- | ----------- | ----- | ---------------------------------------------------------- |
| `after`  | `String`    | Query | 다음 페이지의 요소를 가져오는 커서.                        |
| `before` | `String`    | Query | 이전 페이지의 요소를 가져오는 커서.                        |
| `limit`  | `Integer`   | Query | 응답에 포함된 요소의 최대 개수. 양의 정수 값이어야 합니다. |

</Table>

### [페이지 구분](fetch-partners-pagination)

이 엔드포인트로부터의 결과는 커서를 사용하여 페이징됩니다. 각 페이지는 이전 및 다음 페이지의 위치를 반환합니다. 이를 콜에 포함하여 특정 페이지를 반환할 수 있습니다. 또한 한 페이지에 포함되는 아이템의 수를 제한할 수 있습니다. 기본값 제한은 `50`개 입니다.

각 결과 페이지는 커서 위치가 포함된 "paging" 객체를 포함합니다. 이 커서 위치를 사용하여 페이지를 이동할 수 있습니다. 예를 들어, 첫 페이지에는 다음과 같은 결과가 포함됩니다.

<CodeBlock title="페이징 처리 플로우 예시:">

```json
"paging": {
   "page_size": "50",
   "collection_size": "49",
   "total": "199",
   "cursors": {
      "after": "g2wAAAACYhW1_gxkAANuaWxq",
      "before": null
   },
   "next": "https://api.adjust.com/public/v1/apps/yxs12pfewq/trackers?after=g2wAAAACYhW1_gxkAANuaWxq&limit=50",
   "previous": null
}
```

</CodeBlock>

두 번째 페이지로 이동하려면 `after` 파라미터의 다음 요청에 `after` 값을 추가합니다. 예시:

<CodeBlock title="cURL" highlight="after=g2wAAAACYhW1_gxkAANuaWxq">

```console
$ curl "https://api.adjust.com/public/v1/apps/yxs12pfewq/trackers?after=g2wAAAACYhW1_gxkAANuaWxq"
```

</CodeBlock>

### [응답 파라미터](fetch-partners-response-format)

<Table>

| 필드                | 데이터 유형 | 설명                            | 예시       |

```json
}
```

### [페이지 구분](fetch-sublinks-pagination)

이 엔드포인트로부터의 결과는 커서를 사용하여 페이징됩니다. 각 페이지는 이전 및 다음 페이지의 위치를 반환합니다. 이를 콜에 포함하여 특정 페이지를 반환할 수 있습니다. 또한 한 페이지에 포함되는 아이템의 수를 제한할 수 있습니다. 기본값 제한은 `50`개 입니다.

각 결과 페이지는 커서 위치가 포함된 "paging" 객체를 포함합니다. 이 커서 위치를 사용하여 페이지를 이동할 수 있습니다. 예를 들어, 첫 페이지에는 다음과 같은 결과가 포함됩니다.

<Tabs>
<Tab title="v2" sync="v2">

<CodeBlock title="페이징 처리 플로우 예시:">

```json
"paging": {
   "next": "https://api.adjust.com/public/v2/apps/gwzpeepw8uf8/trackers/abc123/children?cursor=g2wAAAACYhW1_gxkAANuaWxq",
   "cursor": "g2wAAAACYhW1_gxkAANuaWxq"
}
```

</CodeBlock>

두 번째 페이지로 이동하려면 `next` 링크를 따라 다음의 지정된 `cursor` 위치에서 결과를 반환합니다.

<CodeBlock title="cURL" highlight="cursor=g2wAAAACYhW1_gxkAANuaWxq">

```console
$ curl "https://api.adjust.com/public/v2/apps/gwzpeepw8uf8/trackers/abc123/children?cursor=g2wAAAACYhW1_gxkAANuaWxq"
```

</CodeBlock>

</Tab>
<Tab title="v1" sync="v1">

<CodeBlock title="페이징 처리 플로우 예시:">

```json
"paging": {
   "page_size": "50",
   "collection_size": "49",
   "total": "199",
   "cursors": {
      "after": "g2wAAAACYhW1_gxkAANuaWxq",
      "before": null
   },
   "next": "https://api.adjust.com/public/v1/apps/gwzpeepw8uf8/trackers/abc123/children?after=g2wAAAACYhW1_gxkAANuaWxq&limit=50",

}
```

</CodeBlock>

두 번째 페이지로 이동하려면 `after` 파라미터의 다음 요청에 `after` 값을 추가합니다. 예시:
