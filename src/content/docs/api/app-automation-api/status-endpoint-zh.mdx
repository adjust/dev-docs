使用 `/status` 终端，查看新应用的状态。通过该终端返回的状态，您可以了解 `_Template` 模板应用向新应用复制设置的流程情况。

```http
GET https://settings.adjust.com/api/status/{ticket_token}
```

## 调取复制进程

使用该终端返回设置复制流程的状态。使用 [`/app` 终端](/zh/api/app-automation-api/app-endpoint)创建应用时，会收到 `ticket_token`。

<Table>

| 名称             | 数据类型 | 位置          | 描述                                                                |
| ---------------- | -------- | ------------- | ------------------------------------------------------------------- |
| `ticket_token`\* | `String` | 路径 \(Path\) | 您想要检查的工单识别码。在您创建应用时，`/app` 终端会返回该识别码。 |

</Table>

```json
{
   "status": "String"
}
```

### 状态

-  `NEW`：工单已创建，但迁移工作尚未开始。
-  `OPENED`：工单工作已经开始。
-  `FAILED`：工单工作失败。
-  `COMPLETED`：工单工作成功完成。

## 示例

```console
$ curl \
--header "AdjustAuthorization: Token <adjust api token id="sl-md0000000">" \
--header "SignatureAuthorization: Token <adjust signature token id="sl-md0000000">" \
-X GET https://settings.adjust.com/api/status/<ticket_token id="sl-md0000000">
```

```json
{
   "status": "OPENED",
   "events": {
      "successes": [],
      "failures": []
   },
   "partners": {
      "section": "partners",
      "detail": "Could not be start, because {events} sections has failed"
   }
}
```
