使用 Adjust 应用自动化 API \(App Automation API\) 创建并更新应用。借助该套件中的终端，您可以自动化管理应用和应用设置。

您无需在控制面板中分别创建每个应用，只要创建一个模板应用即可。该模板应用中包含推广活动所必需的全部设置。然后，您就可以借助 Adjust 应用自动化 API，基于该模板创建新应用了。

每个新应用都会复制模板应用或您指定的另一款应用的设置。您也可以针对个别应用进行具体设置，覆盖模板应用设置，或附加新的信息。

采用该 API，您只需要为每款新应用创建 1 个请求，省时省力。

[使用案例](use-cases)
-----------------

如果您拥有多款新应用程序，或经常发布新应用和推广活动，那么 Adjust 应用自动化 API 就非常实用，能够省去分别设置每款新应用的额外工作和成本。该 API 能为您带来下列优势：

1. 复制模板应用的设置，简化 **创建** 应用的流程。Adjust 的技术客户专家团队会帮助您设置模板。
2. 简化应用 **更新** 流程。您可以向 API 发送针对具体合作伙伴和应用的设置，随时更新应用。
3. 始终掌握应用创建流程情况。也就是说，您能够准确了解每款应用处于流程的哪一步。
4. 以程式化方式获取应用相关信息。您无需在控制面板中查看每款应用的跟踪链接，使用该 API 就能轻松获得应用信息。

[开始使用](get-started)
-------------------

### [验证请求](authenticate-your-requests)

要使用应用自动化 API，请验证您的调用。请按照[验证指南](/en/api/app-automation-api/authentication)中的说明操作。

### [创建新应用](create-new-apps)

<callout type="note"> 

Adjust 的技术客户专家团队会帮助您设置模板应用。要开始进行设置，请联系您的客户经理。
</Callout> 

向 [`/app`终端](/en/api/app-automation-api/app-endpoint)发送包含应用信息的 POST 请求，创建应用。如果您发送的是新应用，Adjust 就会为其创建条目，并返回新的应用识别码。

发送[针对具体合作伙伴的设置](/en/api/app-automation-api/channel-setup)，以设置合作伙伴模块。

Adjust 会用模板应用的设置填充新应用。

### [查看新应用状态](check-the-status-of-your-new-app)

向 [`/status`终端](/en/api/app-automation-api/status-endpoint)发送 GET 请求，即可查看新应用的状态。通过该操作，您可以看到从模板向新应用复制设置的后台工作状态。

### [获取应用相关信息](get-information-about-your-app)

创建新应用后，您可以向[`/app/{adjust_app_token}`终端](/en/api/app-automation-api/app-endpoint#app-token-parameters)发送 GET 请求，调取关于应用的信息。

您也可以向 [`/app`终端](/en/api/app-automation-api/app-endpoint)发送带有应用商店 ID 和平台的 GET 请求，获取信息。

### [更新应用](update-your-app)

如果您变更了模板应用设置，可以用新信息更新应用。向[`/app`终端](/en/api/app-automation-api/app-endpoint)发送`force_update`请求，传送想要变更的设置，进行应用更新。您可以发送[具体合作伙伴设置](/en/api/app-automation-api/channel-setup)更新应用，以设置合作伙伴模块。
