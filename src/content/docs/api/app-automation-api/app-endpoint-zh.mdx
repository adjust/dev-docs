使用 `/app` 终端自动创建和更新新应用。

创建新应用
-----

使用模板应用的设置创建新应用。如果您的应用已经存在，可以通过 `force_update` 参数更新设置。

```http
POST https://settings.adjust.com/api/app
```

### 参数

<table search> 

|            参数            | 数据类型 | 位置 |                                     描述                                      |
|--------------------------|------|----|-----------------------------------------------------------------------------|
| `store_id`\*            | 字符串  | 正文 | Bundle ID \(Android\) \| Store ID \(Apple\)                             |
| `platform`\*            | 字符串  | 正文 | `android` \| `ios`                                                          |
| `name`\*                | 字符串  | 正文 | 应用的名称。                                                                      |
| `bundle_id`\*           | 字符串  | 正文 | 您应用的 Bundle ID。                                                             |
| `android_signature_hash` | 字符串  | 正文 | 从 keystore 文件中生成的 SHA1 哈希，用来创建 SDK 签名。                                      |
| `channel_setup`          | 对象   | 正文 | 应用合作伙伴模块设置所必需。请参阅[途径设置一文](/en/api/app-automation-api/channel-setup)，了解更多信息。 |
| `editors`                | 字符串  | 正文 | 用户应拥有应用的访问权限的用户邮箱列表。                                                        |
| `template_app_token`     | 字符串  | 正文 | 作为模板的 Adjust 应用识别码。如果您不使用模板应用，该识别码就为必需。                                     |
| `force_update`           | 布尔   | 正文 | 发送该值为 true，以更新现有应用设置。                                                       |

</Table> 

### 响应参数

<table> 

|         参数         | 数据类型 |                                          描述                                          |
|--------------------|------|--------------------------------------------------------------------------------------|
| `adjust_app_token` | 字符串  | Adjust 应用识别码，用于 SDK 集成和其他 API 请求。                                                    |
| `name`             | 字符串  | 应用的名称。                                                                               |
| `sk_app_id`        | 整数   | iOS 应用的 StoreKit 应用 ID。                                                              |
| `store_id`         | 字符串  | Bundle ID \(Android\) \| Store ID \(Apple\)                                      |
| `bundle_id`        | 字符串  | 安卓或 iOS 应用的 Bundle ID                                                                |
| `ticket_token`     | UUID | Ticket ID，用来查看当前工作状态。请参阅[状态终端文档](/en/api/app-automation-api/status-endpoint)，了解更多信息。 |

</Table> 

### 示例

```console
$ curl \
--header "AdjustAuthorization: Token <adjust api token id="sl-md0000000">" \
--header "Content-Type: application/json" \
--data '{ "name":"Test App Name", "bundle_id":"test.bundle.app", "store_id":"test.bundle.app", "platform":"android", "channel_setup": { "facebook": {"app_id": "123456789"} } }' \
-X POST https://settings.adjust.com/api/app
```

```json
{
   "adjust_app_token": "p77yk727r18g",
   "name": "Test App Name",
   "sk_app_id": null,
   "store_id": "test.bundle.app",
   "bundle_id": "test.bundle.app",
   "ticket_token": "af7702a9-ba95-6ba2-bdac-3f83e55ed8d3"
}
```

调取应用详情
------

使用应用商店 ID 和平台名称调取应用详情。

```http
GET https://settings.adjust.com/api/app?store_id={}&platform={}
```

### 参数

<table> 

|      名称       | 数据类型 | 位置 |                       描述                        |
|---------------|------|----|-------------------------------------------------|
| `store_id`\* | 字符串  | 查询 | Bundle ID \(Android\) \| Store ID \(Apple\) |
| `platform`\* | 字符串  | 查询 | `android` \| `ios`                              |

</Table> 

### 响应参数

<table> 

|            名称            | 类型  |                       描述                        |
|--------------------------|-----|-------------------------------------------------|
| `name`                   | 字符串 | 应用名称                                            |
| `platform`               | 字符串 | `android` \| `ios`                              |
| `store_id`               | 字符串 | Bundle ID \(Android\) \| Store ID \(Apple\) |
| `adjust_app_token`       | 字符串 | 被请求应用的识别码。                                      |
| `additional_info`        | 对象  | 终端此前接收的全部应用详情。                                  |
| `channel_setup`          | 对象  | 合作伙伴特定参数，请参考途径设置                                |
| `cross_promo_token`      | 字符串 | 将被用于 CrossPromotion 的 Adjust 跟踪码 \(可用后\)。     |
| `android_signature_hash` | 字符串 | 从 keystore 文件中生成的 SHA1 哈希，用来创建 SDK 签名。          |

</Table> 

### 示例

```console
$ curl \
--header "AdjustAuthorization: Token <adjust api token id="sl-md0000000">" \
-L -X GET 'https://settings.adjust.com/api/app?store_id=test.bundle.app&platform=android'
```

```json
{
   "name": "Test App Name",
   "platform": "android",
   "store_id": "test.bundle.app",
   "adjust_app_token": "p77yk727r18g",
   "additional_info": { "overrides": {} },
   "channel_setup": {
      "facebook": { "app_id": "123456789" },
      "snapchat": { "app_id": "test.bundle.app" }
   },
   "cross_promo_token": "hgj3li1",
   "android_signature_hash": "FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF"
}
```

借助 Adjust 应用识别码调取应用详情
---------------------

使用 Adjust 应用识别码调取应用详情。

```http
GET https://settings.adjust.com/api/app/{adjust_app_token}
```

### 参数

<table> 

|          名称           | 数据类型 |      位置       |                       描述                       |
|-----------------------|------|---------------|------------------------------------------------|
| `adjust_app_token`\* | 字符串  | 路径 \(Path\) | Adjust 应用识别码，可以在 Adjust 控制面板或 /app POST 响应中找到。 |

</Table> 

### 响应参数

<table> 

|            名称            | 类型  |                       描述                        |
|--------------------------|-----|-------------------------------------------------|
| `name`                   | 字符串 | 应用名称                                            |
| `platform`               | 字符串 | `android` 或 `ios`                               |
| `store_id`               | 字符串 | Bundle ID \(Android\) \| Store ID \(Apple\) |
| `adjust_app_token`       | 字符串 | 被请求应用的识别码。                                      |
| `additional_info`        | 对象  | 终端此前接收的全部应用详情。                                  |
| `channel_setup`          | 对象  | 合作伙伴特定参数，请参考途径设置                                |
| `cross_promo_token`      | 字符串 | 将被用于 CrossPromotion 的 Adjust 跟踪码 \(可用后\)。     |
| `android_signature_hash` | 字符串 | 从 keystore 文件中生成的 SHA1 哈希，用来创建 SDK 签名。          |

</Table> 

### 示例

```console
$ curl \
--header "AdjustAuthorization: Token <adjust api token id="sl-md0000000">" \
-X GET https://settings.adjust.com/api/app/p77yk727r18g/
```

```json
{
   "name": "Test App Name",
   "platform": "android",
   "store_id": "test.bundle.app",
   "adjust_app_token": "p77yk727r18g",
   "additional_info": { "overrides": {} },
   "channel_setup": {
      "facebook": { "app_id": "123456789" },
      "snapchat": { "app_id": "test.bundle.app" }
   },
   "cross_promo_token": "hgj3li1",
   "android_signature_hash": "FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF:FF"
}
```

调取应用渠道
------

调取拥有应用活跃跟踪链接的途径列表。

```http
GET https://settings.adjust.com/api/app/{adjust_app_token}/channels
```

### 参数

<table> 

|          名称           | 数据类型 |      位置       |                       描述                       |
|-----------------------|------|---------------|------------------------------------------------|
| `adjust_app_token`\* | 字符串  | 路径 \(Path\) | Adjust 应用识别码，可以在 Adjust 控制面板或 /app POST 响应中找到。 |

</Table> 

### 响应参数

<table> 

|     参数     |  数据类型   |        描述        |
|------------|---------|------------------|
| `channels` | 数组<字符串> | 该应用活跃途径的渠道 slug。 |

</Table> <accordion title="渠道 slug"> 

* `apple_search_ads`
* `applovin`
* `chartboost`
* `crossinstall`
* `crosspromotion`
* `digitalturbine`
* `facebook`
* `gameanalytics`
* `google`
* `ironsource`
* `snapchat`
* `tapjoy`
* `unityads`
* `vungle`
* `inmobi`

</Accordion> 

### 示例

```console
$ curl \
--header 'AdjustAuthorization: Token <adjust api token id="sl-md0000000">' \
--header "SignatureAuthorization: Token <adjust signature token id="sl-md0000000">" \
--X GET 'https://settings.adjust.com/api/app/927t3s86rzsw/channels'
```

```json
[
   "unityads",
   "digitalturbine",
   "jetfuel_api",
   "jetfuel",
   "criteo",
   "tapjoy",
   "bytedance",
   "moloco",
   "adikteev",
   "pinsightmedia",
   "applovin",
   "aura_ironsource",
   "freakout",
   "tradedoubler"
]
```
