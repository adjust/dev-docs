一部のパートナーモジュールについては、パートナー固有のIDをパスする必要があります。アプリオートメーションAPIは、`/app` POSTコールでチャネル情報のオブジェクトをパスすることでこれを可能にします。キーと値の組み合わせを使用して、このオブジェクトで多くのパートナーの情報をパスすることができます。

<Callout type="note">

アプリを設定する際、チャネル設定はオプションです。これは、アプリを作成するまで詳細が入手できない可能性があるためです。

</Callout>

テンプレートアプリにパートナー設定を追加した場合、APIはデフォルトでこれらをコピーしません。パートナー設定を表示させるには、[`/app` POSTコール](/ja/api/app-automation-api/app-endpoint#parameters)に対応するパラメーターを入力する必要があります。

## ワークフロー {#workflow}

アプリ作成時にパートナー固有の情報を追加できます。後から、POSTコールで情報をパスして追加することも可能です。APIがこの情報を受け取ると、APIはパートナー設定を完了しようとします。パートナー設定が完了するまで、APIではクリックとインプレッションURLを利用できません。

1. [`/app` POSTエンドポイント](/ja/api/app-automation-api/app-endpoint#parameters)を使用してアプリを作成します。
2. パートナーIDを受け取ったら、`channel_setup`パラメーターでこのエンドポイントを呼び出します。
3. `/app`エンドポイントを呼び出す度に、全ての`channel_setup`情報を含めます。

## パラメーター {#parameters}

<Callout type="important">

全てのパラメーターを、`channel_setup`オブジェクト内でキーと値の組み合わせとしてパスする必要があります。

</Callout>
<Table>

| パートナー      | 必須パラメーター                                        | 説明                                                                                            |
| --------------- | ------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `snapchat`      | `snap_app_id`                                           | アプリのSnapchat ID                                                                             |
| `facebook`      | `app_id`                                                | アプリのFacebook ID                                                                             |
| `google`        | `link_id`                                               | Google 広告のリンクID                                                                           |
| `gameanalytics` | `key`                                                   | GameAnalyticsの一意のゲームキー                                                                 |
| `tapjoy`        | `app_id`                                                | アプリのTapjoy ID                                                                               |
| `unityads`      | `game_id`                                               | アプリのUnity Ads ID                                                                            |
| `vungle`        | `app_id`                                                | アプリのVungle ID                                                                               |
| `chartboost`    | <ul><li>`app_id`</li><li>`token`</li></ul>              | <ul><li>`app_id`：アプリのChartboost ID</li><li>`token`：アプリのシグネチャー</li></ul>         |
| `inmobi`        | <ul><li>`property_id`</li><li>`advertiser_id`</li></ul> | <ul><li>`property_id`：InmobiでのアプリのID</li><li>`advertiser_id`：Inmobiの広告主ID</li></ul> |

</Table>

## 例 {#example}

以下の例には、利用可能な全てのパートナーが含まれています。

<CodeBlock title="利用可能なパートナー">

```json
"channel_setup": {
  "snapchat": { "snap_app_id": "snapchat app id" },
  "facebook": { "app_id": "facebook_app_id" },
  "google": { "link_id": "adwords_link_id" },
  "gameanalytics": { "key": "unique game key" },
  "tapjoy": { "app_id": "tapjoy_app_id" },
  "unityads": { "game_id": "unity_game_id" },
  "vungle": { "app_id": "vungle_app_id" },
  "chartboost": { "app_id": "chartboost_app_id", "token": "app_signature" },
  "inmobi": { "property_id": "inmobi_gmp_id", "advertiser_id": "adv_id" }
}
```

</CodeBlock>
